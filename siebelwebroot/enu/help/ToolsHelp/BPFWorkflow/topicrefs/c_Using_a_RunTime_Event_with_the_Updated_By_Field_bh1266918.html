Using a Runtime Event with the Updated By Field

If a Workflow Process step includes a runtime event that uses a processing mode that runs locally to start or resume a Workflow Process, then the value in the Updated By field identifies the user who is currently logged into Siebel CRM.

If a Workflow Process runs asynchronously, then it will run as the user defined on the Workflow Process Manager component (typically, SADMIN).
